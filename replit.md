# DentalCAD Studio

Professional admin-only dental CAD application for viewing, analyzing, and exporting intraoral scan files. Built for dental technicians and orthodontists working with Helios 500 and similar scanners.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/dental-cad run dev` — run the frontend (port assigned automatically)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite, Three.js, React Three Fiber, @react-three/drei, Zustand
- API: Express 5 with multer for large file uploads
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (zod/v4), drizzle-zod
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — API contract (source of truth)
- `lib/db/src/schema/scans.ts` — DB schema for scans, measurements, annotations
- `artifacts/api-server/src/routes/` — Express route handlers (scans, measurements, annotations, exports)
- `artifacts/dental-cad/src/modules/` — Modular frontend: renderer, loader, toolbar, measurements, annotations
- `artifacts/dental-cad/src/pages/` — Pages: Viewer (main CAD), Scans (library), Settings
- `artifacts/dental-cad/src/store/viewerStore.ts` — Zustand global viewer state
- `artifacts/api-server/uploads/` — Uploaded scan files stored on disk

## Architecture decisions

- OpenAPI-first: All API contracts defined in openapi.yaml, client hooks and Zod schemas generated via Orval codegen
- File storage: Scan files stored directly on disk in `artifacts/api-server/uploads/` (not object storage) for simplicity; filePath stored in DB
- Mesh stats estimated server-side from binary STL header (triangle count at offset 80) for immediate feedback without parsing the full mesh
- WebGL rendering via React Three Fiber with graceful fallback when GPU is unavailable (Replit sandbox)
- Modular architecture: each 3D capability (rendering, loading, tools, measurements, annotations) lives in its own module directory

## Product

Admin-only dental CAD workstation for:
- Importing STL/OBJ/PLY intraoral scan files (drag-and-drop or browse)
- 3D viewing with orbit/pan/zoom controls, preset views (Occlusal/Front/Left/Right/Bottom)
- Solid, wireframe, and transparency render modes with opacity control
- Measurement tools (distance, angle, area, perimeter, depth)
- Annotation system for labeling anatomical features
- Scan library for managing multiple patient scans
- STL export of loaded scans

## Gotchas

- WebGL is not available in Replit's sandboxed preview iframe — the 3D viewport shows a graceful fallback. It will render correctly in any standard browser.
- The `lib/api-zod/tsconfig.json` must include `"lib": ["esnext", "dom"]` to support File/Blob types generated by the multipart upload schema.
- The `stats` route (`/scans/stats`) must be registered BEFORE the `/:id` route to avoid being caught by the param matcher.
- Multer v2 API is used — `upload.single("file")` middleware pattern.
- Wouter v3 uses `useLocation()` not `useNavigate()` for programmatic navigation.
